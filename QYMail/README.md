# QYMail

### Quantum-Resilient Secure Communication Using Temporal Entropy Fragments

QYMail is a secure communication architecture designed to reduce long-term cryptographic exposure in email systems. It introduces a new concept called **Temporal Entropy Fragment (TEF)**, where encryption keys depend on **short-lived physical entropy generated by hardware**.

The system ensures that the ability to decrypt encrypted communication **naturally disappears after a limited time window**, even if encrypted data is stored by attackers.

# Introduction

Modern digital communication systems rely on cryptographic protocols to protect sensitive data transmitted over networks. Secure email systems used by governments, banks, and enterprises typically rely on:

* Public Key Infrastructure (PKI)
* TLS encryption
* RSA or ECC key exchange
* Symmetric encryption such as AES

These technologies provide strong protection today, but they still depend on **long-term cryptographic keys and stored encrypted data**.

If cryptographic algorithms are broken in the future or private keys are compromised, previously captured encrypted communication may become vulnerable.

QYMail introduces a new approach where encryption keys are partially derived from **short-lived physical entropy**, making long-term decryption significantly more difficult.

---

# The Problem with Current Secure Email Systems

Secure email systems typically operate using the following structure:

```
Client → Secure TLS Connection → Mail Server → Message Storage
```

Security depends on several components:

* Server private keys
* Certificate authorities
* Key management systems
* Symmetric encryption for message protection

Although these systems provide strong security, they have an important limitation.

Encrypted messages may remain stored for many years. If an attacker records encrypted communication today, they may attempt to decrypt it later when new computational capabilities or cryptographic breakthroughs become available.

This threat is commonly known as:

**Harvest Now – Decrypt Later**

It is particularly relevant in the context of:

* Quantum computing research
* Long-term data storage
* Advanced cryptanalysis
* Key database breaches

---

# Existing Solutions and Their Limitations

Several security mechanisms attempt to reduce long-term cryptographic risk.

### Forward Secrecy

Forward secrecy generates temporary session keys during communication.

However, encrypted messages may still be stored indefinitely. If the session keys are somehow recovered or cryptographic weaknesses appear later, stored ciphertext may still become vulnerable.

---

### Key Destruction

Some systems delete encryption keys after communication.

While this reduces the risk of key compromise, the encrypted data itself still remains stored. Future breakthroughs may still expose the data.

---

### Hardware Security Modules (HSMs)

HSMs protect sensitive keys inside secure hardware.

However, HSMs still rely on **stored secrets**, which remain long-term targets for attackers.

---

# QYMail Overview

QYMail introduces a new secure communication architecture that combines traditional cryptography with **temporary physical entropy**.

Instead of generating encryption keys only from stored secrets, QYMail generates keys using:

* a protected device secret stored inside hardware
* a **Temporal Entropy Fragment (TEF)** generated during communication

Because the entropy fragment exists only briefly, the same encryption key **cannot be regenerated later**.

This significantly reduces the risk that stored encrypted communication could be decrypted in the future.

---

# Core Innovation: Temporal Entropy Fragment (TEF)

A **Temporal Entropy Fragment** is a small piece of entropy derived from real-time physical noise inside hardware.

Examples of physical entropy sources include:

* Thermal noise in electronic circuits
* Oscillator phase jitter
* RF noise
* SRAM startup behavior
* Capacitor charge variations

These signals naturally fluctuate due to physical processes and are difficult to predict or reproduce.

Key properties of TEF:

* Generated from **physical hardware noise**
* Exists only during a communication session
* Not permanently stored
* Cannot be recreated once the hardware state changes

This entropy fragment is combined with a secure device secret to generate a temporary encryption key.

Once the communication session ends, the entropy fragment disappears and the key is erased.

# QYMail System Architecture

The QYMail system integrates with existing email infrastructure while introducing secure hardware entropy generation.

Main components:

### Client Device

Users send and receive messages using standard email clients.

---

### QYMail Server

The server manages secure communication sessions and message routing.

---

### Secure Hardware Module

A protected hardware module performs cryptographic operations and stores sensitive secrets.

It contains:

* device secret
* root identity key
* entropy generation hardware

---

### Hardware Entropy Generator

Electronic circuits generate random signals using physical noise sources such as:

* thermal noise
* oscillator jitter
* RF noise

These signals are used to derive the Temporal Entropy Fragment.

---

### Ephemeral Key Generation

The device secret and TEF are combined to generate a temporary encryption key used during the communication session.

---

# End-to-End Workflow

1. A client initiates a secure connection to the QYMail server.

2. The secure hardware module samples physical entropy from the hardware entropy generator.

3. A Temporal Entropy Fragment is derived from the sampled physical noise.

4. The device secret and entropy fragment are combined to generate a temporary encryption key.

5. The temporary key is used to establish a secure communication session.

6. Messages are encrypted using secure symmetric encryption during transmission.

7. When the communication session ends:

   * the temporary key is erased
   * the entropy fragment disappears as the hardware state changes.

Because the entropy fragment no longer exists, the same encryption key **cannot be regenerated later**.

# Security Advantages

QYMail introduces several important security improvements.

### Time-Bound Decryption Capability

Decryption is only possible during the short period when the entropy fragment exists.

---

### Reduced Long-Term Cryptographic Exposure

Even if encrypted messages are captured and stored, they cannot be decrypted later if the required entropy fragment no longer exists.

---

### Hardware-Anchored Security

Key generation occurs inside secure hardware modules, reducing exposure to software attacks.

---

QYMail can integrate with existing email systems and network infrastructure.
